# 泰国蜜月旅行规划工作区：第一版设计规格

## 目标

建立一个可持续迭代的泰国蜜月旅行规划工作区：结构化文件供 agent 保存事实、来源、偏好与决策；静态 HTML 页面供两位旅行者查看路线、住宿候选和下一步决策。第一版只建立可信的事实底座与少量真实住宿候选，不提前形成大而全的最终攻略。

## 已批准的产品方向

工作区采用“证据优先的蜜月手账”结构。信息明确区分为已确认事实、根据事实推导的信息、带核验时间的网页价格快照、尚待确认的偏好和已作出的决定。视觉采用温暖克制的旅行手账与精品酒店编辑风，避免普通后台仪表盘样式。

住宿预算采用“分段花钱”：曼谷优先交通和位置；芭提雅与象岛可以为海景、沙滩、度假氛围和蜜月体验提高预算。这个原则是候选筛选规则，不是固定的每日金额上限。

## 第一版信息架构

智能体数据分为四类：行程事实与推导、住宿候选与网页来源、偏好与决策、研究日志。每条住宿价格必须记录目的地、入住和退房日期、2 位成人、房型、晚数、税费口径、取消政策、核验时间、来源链接与可靠性备注。无法确认的字段保留为空并标注原因，不以猜测补齐。

HTML 看板包含五个区域：顶部路线总览；四段住宿任务；当前少量候选；预算分配逻辑；下一步唯一决策。每个候选展示适合原因、主要取舍、价格快照、核验状态和预订页链接。第一版每段最多 3 个候选，优先形成有差异的选择，而不是堆积相似酒店。

## 第一轮住宿研究范围

第一段曼谷优先 Airbnb，并围绕 Sala Daeng、Silom、Lumphini、Siam 与 Ratchathewi 等交通方便区域形成少量候选；如果 Airbnb 页面无法稳定核验完整报价，允许记录房源页与区域信息，同时明确把总价标为待用户在页面复核。第二段芭提雅比较市中心便利、安静海滩度假与公寓性价比。第三段象岛比较 Klong Prao、Kai Bae 等适合两人度假的海滩区域。第四段曼谷只有 1 晚且次日 12:35 起飞，优先降低次日交通压力；机场未出现在截图中，后由用户确认为素万那普机场（BKK），因此第一版可以围绕 Airport Rail Link 形成候选。

首轮候选不是最终推荐。研究目标是用真实差异引导下一轮偏好，例如“更看重私密海滩还是步行餐饮”“愿意为独立泳池或海景房增加多少预算”。

## 价格与来源规则

网页价格属于核验时刻的快照，不保证持续有效。页面显示的原价、促销价、会员价、含税总价与未含税价格不得混称。只有页面明确匹配日期、2 位成人、1 间房和正确晚数时，才能把数值展示为当前报价；否则展示为参考价或待复核。

看板中的每个外部事实应保留可打开的直接来源。搜索摘要可以用于发现候选，但不能单独作为精确报价依据。Booking、Airbnb 等页面的动态内容、登录价格与库存可能变化，看板需显示最后核验日期。

## 文件结构

`work/` 保存智能体使用的数据、研究记录与中间材料；`outputs/` 保存用户直接打开的静态 HTML 看板及必要的本地展示资源。原始截图的工作副本保存在 `work/`，并在数据中记录它是事实来源。第一版不引入构建工具、数据库或服务器，HTML 可直接双击打开。

## 交互与视觉

看板使用桌面与手机均可阅读的响应式布局。首屏立即呈现路线、11 晚分配、已确认决定和下一步唯一问题；住宿卡片支持按目的地浏览，并以视觉标签区分“已核验报价”“参考信息”“待复核”。页面不收集数据、不提交表单，也不依赖登录状态。

视觉方向使用泰国热带暮色、象牙纸与深墨绿色作为主色，搭配少量藏红花色强调；字体、留白和细节强调旅行编辑感。动效克制，只用于首屏进入和卡片状态反馈，并尊重减少动态效果的系统设置。

## 验收标准

行程日期与 11 晚能够逐段对账；截图中未提供的时间在第一版标记为待规划，不会伪装成已确定事实；机场在得到机票信息前不会被补造；每个精确报价均匹配正确日期、2 位成人、1 间房和晚数；每段住宿最多 3 个第一轮候选；候选来源可打开且显示核验日期；价格状态不会把参考价误称为实价；HTML 可离线打开，桌面和手机布局均可阅读；页面清楚指出下一轮只需要用户决定的一项偏好。

## 第一版之后的迭代边界

用户每回答一个偏好问题，智能体更新决策记录、候选状态和 HTML 看板。只有住宿方向逐步收敛后，才开始把每日观光、餐饮与休息节奏组合成 2 至 3 套完整方案。机票和已定交通不在第一版中重新规划。

## 项目语言与币种规则

项目级规划、提问、研究、展示、语言、币种与记忆边界统一由 `.agents/skills/plan-trips-iteratively/SKILL.md` 维护，本设计文档不再复制这些工作方法。当前看板采用人民币主价格；本轮曼谷 Airbnb 报价按 `1欧元 = 8人民币` 换算，原始欧元价仍保留在结构化数据中。
